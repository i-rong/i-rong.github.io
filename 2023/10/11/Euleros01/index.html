宏 含义
MODULE_LICENSE 声明代码采用的开源协议，由于linux内核代码采用GPL协议，而GPL协议要求使用GPL协议的代码同样需要使用GPL协议，即代码开源。被内核接受的有意义的许可证有 GPL，GPL v2，GPL and additional rights，Dual BSD/GPL，Dual MPL/GPL，Proprietary。否则，收到内核被污染“kernel tainted”的警告。
module_param typesafe helper for a module/cmdline parameter。提供一种类型安全的命令行参数。 @value: the variable to alter, and exposed parameter name. @type: the type of the parameter. @perm: visibility in sysfs. eg. 0444 for world-readable. 0644 for root-writable. 这里其实和linux文件权限一致。
MODULE_PARM_DESC 使用modinfo时，可以显示出参数的描述。One for each parameter, describing how to use it. Some files do multiple of these per line, so can’t just use MODULE_INFO.
module_init 模块初始化入口。driver initialization entry point.
module_exit 模块出口。driver exit entry point.
__init kernel macros. 标识函数为初始化函数。在模块装载之后，模块装载就会将初始化函数扔掉。这样可以将该函数占用的内存释放出来。 Detail in __init & __exit
__exit kernel macros. 标识函数为退出函数，在模块卸载时执行。 Detail in __init & __exit

[1] MODULE_LICENSE // 在源码下的include/linux/module.h文件中

[2] module_param // 在源码下的include/linux/module.h文件中

[3] MODULE_PARAM_DESC

[4] module_init & module_exit

[5] __ init& __exit

三、 遇到的问题

在我将名字中间加入空格的时候会报错：

参考这篇博客得以解决https://www.javaer101.com/ja/article/14914575.html

当运行insmod hello_magic_student.ko id=”22920202204632” name=”Yao huaiyu” age=21时，shell会使用引号，并且insmod二进制文件将字符串name=Yao huaiyu作为参数。它将原样传递给内核，然后全部传递给内核parse_args函数(在kernel/params.c中)，该函数依次调用next_arg以将下一个参数拆分为名称和值。

改为insmod hello_magic_student.ko id=”22920202204632” name=’”Yao huaiyu”’ age=21

注意这里在双引号外面多加了单引号。

再次运行make run

可以看到，程序正常执行。
